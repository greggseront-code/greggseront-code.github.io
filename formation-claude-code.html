Fil rouge

Un projet existant, une feature à ajouter

Les participants arrivent sur une base de code qu’ils ne connaissent pas, comme en entreprise. La journée suit le chemin complet : comprendre ce qui existe, poser le cadre, spécifier et implémenter une feature de façon itérative, avec review et tests à chaque étape.

Le projet fil rouge est en TypeScript, backend Node.js. Il n’est pas nécessaire de maîtriser ce stack : apprendre à travailler avec Claude Code, c’est précisément apprendre à naviguer dans un projet sans tout connaître d’avance. La méthode, le cadrage et les réflexes acquis s’appliquent à n’importe quel stack.

Programme

Déroulé de la journée

Le programme suit un vrai cycle de développement : exploration, cadrage, spécification, implémentation, review et validation.

1. Explorer la base de code avec Claude Code

Comprendre rapidement un projet inconnu : structure, conventions, dépendances et zones sensibles. Claude Code devient un outil d’onboarding technique.

2. Poser le cadre : CLAUDE.md et skills

À partir de ce qui a été découvert, écrire un CLAUDE.md utile et configurer les skills adaptés au stack et aux contraintes du projet : ce qu’on délègue, ce qu’on garde, comment on travaille ensemble.

CLAUDE.md skills/ rules

3. Spécifier la feature et son architecture

Clarifier le besoin, faire apparaître les ambiguïtés, rendre visibles les choix de conception avant de toucher au code. Ne pas laisser Claude Code décider seul de l’architecture.

4. Implémenter de façon itérative

Construire par petits incréments. À chaque étape : review du code généré, identification des dérives, correction du cadre si nécessaire.

5. Tester et valider

Écrire et relire les tests avec Claude Code. Ajouter les cas limites. Décider explicitement ce qu’on accepte, pas seulement ce qui compile.

Pour les participants qui avancent plus vite : tests d’interface.
